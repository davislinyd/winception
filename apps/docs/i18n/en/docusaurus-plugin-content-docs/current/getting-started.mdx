# Getting Started

Winception 2.0 is a single-host, on-premises product. Management binds to `127.0.0.1:8080` by default; PXE DHCP, TFTP and HTTP form a separate data plane.

## Familiar v1 workflow

v2's **Deploy / Monitor** preserves v1's familiar order: inspect Profile, OS Image, Endpoint and HTTP/TFTP/DHCP/Torrent state in Deploy, then track clients through Fleet cards, stage flow, evidence and the torrent tracker in Monitor. Buttons still create asynchronous v2 operations: the UI first shows the accepted operation ID, then running/succeeded/failed state; `409 OPERATION_CONFLICT` lists the conflicting operation and resource. The `deployment.changed` SSE event only tells the browser to retrieve a REST snapshot; it never carries raw Fleet, log or evidence data. Torrent Extend/Continue is invoked from the corresponding row, without manually entering a run ID or client ID.

## Acceptance scope

1. A fresh Windows 11 Pro or Enterprise x64 Generation 2 VM with nested virtualization and two NICs.
2. Fresh install → repair → uninstall with State preservation → reinstall.
3. One Generation 2 Secure Boot client with at least 4 GB fixed RAM through `windows-desktop-ready`.
4. One Software Test with ingress stopped and Fleet empty, followed by checkpoint restore.

A single-client run is not evidence for two four-client rounds or a physical laptop. Until this scope passes, the build remains an internal prerelease.

## Required files

- `Winception-v2.msi`
- `Winception-Local-CodeSigning.cer`
- `Install-Winception.ps1`
- `release-manifest.json` and `SHA256SUMS`
- Elevated PowerShell and at least 2 GB free on the system drive
- An Internet-capable management NIC and an isolated PXE NIC with no other DHCP responder

Complete [VM topology](/docs/vm-topology) before using the [interactive installer](/docs/install). The installer does not change NICs or start DHCP.

## Definition of done

Source tests, an MSI hash or Web health alone do not complete VM acceptance. Evidence must identify the exact Git commit, MSI SHA-256, installed VM, PXE client and Software Test run.
