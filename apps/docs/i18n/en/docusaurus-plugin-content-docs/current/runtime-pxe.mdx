# Runtime, Image, Profile and Isolated PXE

<FlowExplorer />

## Required sequence

1. Read the project root, interface, IP, DHCP range, router, HTTP base, boot mode, OS image and profile from live Web or API state.
2. Select the PXE NIC and endpoint; confirm management and PXE subnets do not overlap.
3. Prepare runtime, select or import an OS image, and publish a deployment profile.
4. Run preflight. Warnings may remain and must be recorded; blocking failures must be zero.
5. Start DHCP, TFTP and HTTP only after confirming no other DHCP responder exists on the isolated network.
6. Boot a Generation 2 Secure Boot client over UEFI IPv4 PXE and track it through `windows-desktop-ready`.
7. Preserve run evidence and stop ingress.

## Guided Setup in Deploy

The right-side Guided Setup lets the operator select the current step and exposes only its mapped safe v2 action. When the ingress gate appears, select **Run preflight**. Until blocking failures are cleared, Start all and stopped HTTP, TFTP, DHCP and Torrent start controls remain disabled. It never starts services automatically.

## Boundary

Offline ISO is an additive snapshot and does not replace PXE acceptance. A Git clone is not deployment runtime; never patch the Web-selected runtime root manually.
