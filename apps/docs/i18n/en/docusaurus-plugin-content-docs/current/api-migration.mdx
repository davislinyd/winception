# API, Events and v1 Migration

## `/api/v2`

Requests, responses, params and query strings use complete JSON Schema. Errors contain `error.code`, `message`, optional `correctiveAction` and `correlationId`; raw PowerShell and secrets are never returned.

Mutations return an operation ID. Fleet, operation and state updates use versioned SSE. After a disconnect, load a REST snapshot before resuming the event stream. OpenAPI is available only to an authenticated management session.

## v1 importer

The importer supports dry-run, backup, rerun and an explicit migration report. It imports config, profiles, software/custom-script metadata and re-encrypted secrets, but excludes running operations, DHCP leases, service PIDs and locks.

The standard MSI detects `C:\OSDCloud\HostTools\App` and `State`. When v1 is present, import runs before services start. Non-secret sources and the report are retained under `C:\ProgramData\Winception\State\migration-backups\<fingerprint>`. Plaintext secrets are never copied into that backup; they are converted directly to LocalMachine DPAPI ciphertext, and the original v1 State is not modified.

```powershell
& "$env:ProgramFiles\Winception\node\node.exe" `
  "$env:ProgramFiles\Winception\app\dist\v2\node\apps\agent\src\migrateV1.js" `
  --app-root C:\OSDCloud\HostTools\App `
  --state-root C:\OSDCloud\HostTools\State `
  --v2-state-root "$env:ProgramData\Winception\State" `
  --dry-run
```

A dry-run may initialize an empty target SQLite schema, but it writes no migration marker, product data, payload, backup or report. The same fingerprint is safely rerunnable. If the source changes after a successful import, the tool refuses to overwrite the existing v2 State.

After import, Agent rebuilds runtime with a short-lived plaintext projection before opening its named pipe, then scrubs plaintext and clears the rebuild marker. If import, runtime rebuild or the Web health probe fails, MSI restores the v2 database and compatibility projection, preserves v1 State and the migration backup, and does not commit the cutover. Existing evidence remains a read-only archive.
