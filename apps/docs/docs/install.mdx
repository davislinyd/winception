## 安全行為

`Check` 是預設且不修改系統。`Install` 接受「本機 MSI＋CER」或「GitHub prerelease tag」，兩者互斥。所有 `msiexec` 動作使用 `/norestart` 並保存 verbose log。

自簽 CER 只有在使用者先核對 subject、thumbprint、Code Signing EKU、有效期與 manifest，再明確加入 `-TrustSelfSignedCertificate` 時，才匯入 `LocalMachine\Root` 與 `LocalMachine\TrustedPublisher`。

在全新 VM 尚未信任 CER 時，`Check` 仍會先核對 manifest SHA-256、signer thumbprint、Code Signing EKU、有效期與 certificate chain。只有預期自簽 root 尚未受信任可以通過；`HashMismatch`、signer mismatch 或其他 chain error 一律阻擋。

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\Install-Winception.ps1 -Action Check `
  -MsiPath .\Winception-v2.msi `
  -CertificatePath .\Winception-Local-CodeSigning.cer `
  -ReportPath .\winception-check.json

.\Install-Winception.ps1 -Action Install `
  -MsiPath .\Winception-v2.msi `
  -CertificatePath .\Winception-Local-CodeSigning.cer `
  -TrustSelfSignedCertificate -ShowSetupCode -OpenBrowser `
  -ReportPath .\winception-install.json
```

Setup code 只顯示在目前 console，不寫入 MSI log、JSON report 或 diagnostics。

## Fresh install 驗證

- `Winception.Agent`：Running、LocalSystem。
- `Winception.Web`：Running、LocalService、service SID unrestricted。
- `C:\ProgramData\Winception\State\winception-v2.db` 存在且 ACL 正確。
- protected named pipe 可用。
- `http://127.0.0.1:8080/api/v2/health` 回傳正常。
