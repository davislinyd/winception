# Winception v2.0.0-alpha.5 prerelease notes

Release channel: **internal test prerelease for installed-runtime acceptance only**.

This candidate replaces `v2.0.0-alpha.4` because alpha 4 required `node`, `npm`, `OSD` and `OSDCloud` to be independently installed on the target VM before `Prepare runtime`. Alpha 5 uses the fixed Node 24 runtime already delivered by the MSI, removes the target-host npm dependency, and bundles exact OSD 26.4.23.1 and OSDCloud 26.4.17.1 PowerShell Gallery packages after SHA-256 verification. It also registers the IPv4 format in Agent IPC, prepends the packaged module root to every Agent child process, fills missing download metadata for bundled software after v1 migration without overwriting user-defined values, owns interface/subnet-scoped Windows Firewall rules for DHCP, TFTP, HTTP and torrent traffic, and exposes cached OS image annotations through the validated v2 API contract. The DHCP receiver binds to `0.0.0.0`; a second socket bound to the configured PXE IP sends one limited-broadcast reply so multi-NIC routing cannot leak the reply through the management adapter and pre-address UEFI clients can accept it. Server-mode offers provide Options 66/67 without advertising Option 60 or ProxyPXE menus, preventing firmware from switching to an unavailable BINL service on UDP 4011. Boot-config decrypts DPAPI secrets on demand in Agent memory and fails closed with a redacted 503; no persistent plaintext sidecar is required while ingress runs. Deployment ingress cannot start until preflight has run with zero blocking failures. Software Test preserves singleton `steps` as an API array, waits for the SYSTEM installer task to actually start, and writes raw runner failures only to the local diagnostic. Full uninstall removes product firewall rules; a major upgrade preserves them. Its final MSI `ProductVersion` is `2.0.18`, which is higher than all earlier test packages and therefore exercises the actual MajorUpgrade path.

The bundled module versions and hashes are recorded in `THIRD-PARTY-NOTICES.md` and the CycloneDX SBOM. Their complete GPL-3.0 license text is included in the installed module and license directories. Windows ADK and the matching WinPE add-on remain product-managed prerequisites installed by `Prepare runtime` from Microsoft-signed installers.

This build must pass local source/package gates, upgrade the prepared Windows 11 Pro VM, and complete `Prepare runtime` before it may be published. One isolated Generation 2 Secure Boot PXE client and one Software Test with ingress stopped, Fleet empty and checkpoint restore remain required deployment evidence. This prerelease is not production-ready.

## Installation

Download `Install-Winception.ps1` from this release. Do not pipe remote content to PowerShell.

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\Install-Winception.ps1 -Action Check -ReleaseTag v2.0.0-alpha.5 -ReportPath .\check.json
.\Install-Winception.ps1 -Action Install -ReleaseTag v2.0.0-alpha.5 -TrustSelfSignedCertificate -ShowSetupCode -OpenBrowser -ReportPath .\install.json
```

The management UI remains loopback-only during first acceptance. The installer does not change NIC settings or start PXE DHCP. The self-signed certificate is imported only when `-TrustSelfSignedCertificate` is explicitly present.
