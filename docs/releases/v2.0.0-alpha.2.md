# Winception v2.0.0-alpha.2 prerelease notes

Release channel: **internal test prerelease for fresh-VM acceptance only**.

This release replaces `v2.0.0-alpha.1` for installation testing. Alpha 1 incorrectly treated a valid self-signed MSI on a clean host as damaged when Windows reported that its root certificate was not yet trusted. Alpha 2 permits only the expected signer and a chain whose sole trust issue is the expected self-signed root; exact SHA-256 validation still runs first, and signer mismatch, invalid chain status and `HashMismatch` remain blocking.

Fresh install, repair, uninstall with State preservation, reinstall persistence, one isolated Generation 2 Secure Boot PXE client to `windows-desktop-ready`, and one Software Test with ingress stopped, Fleet empty and checkpoint restore remain required acceptance evidence. This prerelease is not production-ready.

## Installation

Download `Install-Winception.ps1` from this release. Do not pipe remote content to PowerShell.

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\Install-Winception.ps1 -Action Check -ReleaseTag v2.0.0-alpha.2 -ReportPath .\check.json
.\Install-Winception.ps1 -Action Install -ReleaseTag v2.0.0-alpha.2 -TrustSelfSignedCertificate -ShowSetupCode -OpenBrowser -ReportPath .\install.json
```

The management UI remains loopback-only during first acceptance. The installer does not change NIC settings or start PXE DHCP. The self-signed certificate is imported only when `-TrustSelfSignedCertificate` is explicitly present.
