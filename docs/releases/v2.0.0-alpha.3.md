# Winception v2.0.0-alpha.3 prerelease notes

Release channel: **internal test prerelease for fresh-VM acceptance only**.

This release replaces `v2.0.0-alpha.2` for installation testing. On a fresh Windows VM, the machine-level `TrustedPublisher` registry store key may not exist. Alpha 2 could import the expected certificate into `LocalMachine\Root` and then fail with `E_ACCESSDENIED` before the MSI started. Alpha 3 initializes the missing store key, resumes safely from the partial Root-only state, imports the same expected thumbprint into `LocalMachine\TrustedPublisher`, and verifies both stores.

The bootstrap report now records the failed stage and error identifier. Exact SHA-256, signer, certificate validity and chain checks still run before trust changes; signer mismatch, invalid chain status and `HashMismatch` remain blocking.

Fresh install, repair, uninstall with State preservation, reinstall persistence, one isolated Generation 2 Secure Boot PXE client to `windows-desktop-ready`, and one Software Test with ingress stopped, Fleet empty and checkpoint restore remain required acceptance evidence. This prerelease is not production-ready.

## Installation

Download `Install-Winception.ps1` from this release. Do not pipe remote content to PowerShell.

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\Install-Winception.ps1 -Action Check -ReleaseTag v2.0.0-alpha.3 -ReportPath .\check.json
.\Install-Winception.ps1 -Action Install -ReleaseTag v2.0.0-alpha.3 -TrustSelfSignedCertificate -ShowSetupCode -OpenBrowser -ReportPath .\install.json
```

The management UI remains loopback-only during first acceptance. The installer does not change NIC settings or start PXE DHCP. The self-signed certificate is imported only when `-TrustSelfSignedCertificate` is explicitly present.
