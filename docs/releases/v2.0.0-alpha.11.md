# Winception 2.0.0-alpha.11 internal candidate

Alpha.11 replaces the unpublished alpha.10 candidate after installed v1 migration showed that Web health could succeed while the privileged Agent was still rebuilding runtime and had not created its named pipe.

The MSI health transaction now requires Web health, a running Agent service and the protected Agent pipe. It fails immediately if Agent stops and waits up to 15 minutes while a valid WinPE rebuild is still progressing. The candidate uses MSI `ProductVersion 2.0.26`, `AGPL-3.0-only`, and the explicitly trusted self-signed test certificate. It remains internal until exact-MSI migration success and rollback are both recorded.
