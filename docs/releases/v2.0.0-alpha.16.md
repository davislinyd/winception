# Winception 2.0.0-alpha.16 internal candidate

Release channel: **unpublished internal candidate**.

Alpha 16 corrects the alpha.15 familiar-UX Guided Setup regression without changing any v2 privilege boundary. The rail now selects a live initialization step and maps it to its existing schema-validated v2 control. When preflight is missing or blocking, both the rail and ingress gate provide `Run preflight`; stopped HTTP, TFTP, DHCP and Torrent starts remain disabled. Fastify, authenticated named-pipe IPC, schemas, OperationCoordinator, SQLite and DPAPI remain unchanged.

The MSI `ProductVersion` is `2.0.31`, above alpha.15's `2.0.30`, solely to permit a controlled in-place v2 upgrade on the OSD Server VM. Reuse the existing checkpoint, keep ingress stopped, verify both Windows services and loopback health after installation, and confirm the Guided Setup preflight action is available before attempting any deployment service start.

No prerelease, GitHub Pages update, tag or `master` merge is authorized. This file records candidate intent only; exact MSI hash and installed-host evidence are added only after local packaging and VM verification.
