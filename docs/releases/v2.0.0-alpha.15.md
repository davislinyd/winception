# Winception 2.0.0-alpha.15 internal candidate

Release channel: **unpublished internal candidate**.

Alpha 15 rebuilds the React management surface around the familiar v1 Deploy / Monitor workflow without changing the v2 privilege boundary. It adds readable service cards, Fleet/evidence detail, torrent tracker row actions, operation-state feedback, evidence-free `deployment.changed` SSE recovery, and bounded torrent extend payloads. Fastify, authenticated named-pipe IPC, schemas, OperationCoordinator, SQLite and DPAPI remain unchanged as security boundaries.

The MSI `ProductVersion` is `2.0.30`, above alpha.14's `2.0.29`, solely to permit a controlled in-place v2 upgrade on the OSD Server VM. Before that upgrade, record a hash-only baseline of the v1 App/State roots, ensure Fleet is empty and ingress is stopped, and create a Hyper-V checkpoint. The test must prove v1 hashes unchanged, installed Web/Agent health, migration preservation, and the familiar UI; it must not claim PXE or physical-laptop acceptance until those paths are run.

No prerelease, GitHub Pages update, tag or `master` merge is authorized. This file records the candidate intent only; exact MSI hash and installed-host evidence are added only after local packaging and VM verification.
