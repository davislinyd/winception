# Winception v2.0.0-alpha.4 prerelease notes

Release channel: **internal test prerelease for fresh-VM acceptance only**.

This release replaces `v2.0.0-alpha.3` for installation testing. Alpha 3 successfully initialized and populated the missing machine-level `TrustedPublisher` store on the fresh VM, but the MSI then rolled back with error 1722/1603 before creating services. WiX expands the directory-valued `APPFOLDER` property with a trailing separator; quoting it directly produced an invalid `-AppRoot "C:\Program Files\Winception\app\"` custom-action argument.

Alpha 4 passes `[APPFOLDER].` to `ProvisionServiceSettings`, preserving the intended app directory without a quote-adjacent trailing separator. Loopback provisioning and its post-install health probe also guard optional TLS settings before reading them under PowerShell `StrictMode`, so the default loopback install does not fail when no TLS object exists. Packaging regression gates block these defects. The exact-hash, signer, self-signed store initialization and stage-qualified report protections from alpha 3 remain enabled.

Fresh install, repair, uninstall with State preservation, reinstall persistence, one isolated Generation 2 Secure Boot PXE client to `windows-desktop-ready`, and one Software Test with ingress stopped, Fleet empty and checkpoint restore remain required acceptance evidence. This prerelease is not production-ready.

## Installation

Download `Install-Winception.ps1` from this release. Do not pipe remote content to PowerShell.

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\Install-Winception.ps1 -Action Check -ReleaseTag v2.0.0-alpha.4 -ReportPath .\check.json
.\Install-Winception.ps1 -Action Install -ReleaseTag v2.0.0-alpha.4 -TrustSelfSignedCertificate -ShowSetupCode -OpenBrowser -ReportPath .\install.json
```

The management UI remains loopback-only during first acceptance. The installer does not change NIC settings or start PXE DHCP. The self-signed certificate is imported only when `-TrustSelfSignedCertificate` is explicitly present.
