# Winception v2.0.0-alpha.7 candidate notes

Release channel: **unpublished internal test candidate**.

Alpha 7 includes the alpha.6 diagnostics/evidence export fixes and closes the management HTTPS rollback failure found during installed VM acceptance. Invalid certificate provisioning now restores Agent and Web in dependency order with bounded retries. Returning to loopback removes exported TLS PFX/CER files so obsolete private-key payloads are not left readable by the Web service.

The candidate uses MSI `ProductVersion 2.0.22`, the existing explicitly trusted self-signed test certificate, and `AGPL-3.0-only`. It must pass local source/package gates, installed upgrade, HTTPS fail-closed/restore, and remaining product acceptance before publication. Alpha.5 remains immutable and available as the last published internal prerelease; alpha.6 was never published.
