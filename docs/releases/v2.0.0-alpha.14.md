# Winception 2.0.0-alpha.14 internal candidate

Atomic deployment progress and metadata writers now retry transient Windows sharing violations for up to four seconds while preserving replace semantics. This closes the reader/writer race discovered during alpha.13 live PXE acceptance and keeps desktop-ready fail-closed.

The candidate retains the monotonic WinPE torrent seed timer, uses MSI `ProductVersion 2.0.29`, `AGPL-3.0-only`, and the explicitly trusted self-signed test certificate. Local gates and two consecutive clean two-client PXE rounds are required; physical-laptop acceptance remains separate.
