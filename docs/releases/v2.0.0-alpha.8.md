# Winception v2.0.0-alpha.8 candidate notes

Release channel: **unpublished internal test candidate**.

Alpha 8 includes the alpha.6 diagnostics/evidence export fixes and closes management HTTPS failures found during the unpublished alpha.7 installed VM build. Child PowerShell failures are reduced to explicit exit codes so transaction cleanup always runs, invalid certificate provisioning restores Agent and Web in dependency order with bounded retries, and the pinned local HTTPS probe bypasses system proxy configuration. Returning to loopback removes exported TLS PFX/CER files. Major upgrades preserve existing protected tokens and management endpoint settings.

The candidate uses MSI `ProductVersion 2.0.23`, the existing explicitly trusted self-signed test certificate, and `AGPL-3.0-only`. It must pass local source/package gates, installed upgrade, HTTPS fail-closed/restore, and remaining product acceptance before publication. Alpha.5 remains the latest published internal prerelease; alpha.6 and alpha.7 were never published.
