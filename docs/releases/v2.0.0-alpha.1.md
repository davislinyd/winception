# Winception v2.0.0-alpha.1 prerelease notes

Status: **not published; internal acceptance candidate only**.

This prerelease may be created only after the exact MSI passes elevated fresh install, repair, uninstall with State preservation, reinstall persistence, one isolated Generation 2 Secure Boot PXE client to `windows-desktop-ready`, and one Software Test with ingress stopped, Fleet empty and checkpoint restore.

## Candidate assets

- `Winception-v2.msi`
- `Winception-Local-CodeSigning.cer`
- `Install-Winception.ps1`
- `release-manifest.json`
- `SHA256SUMS`
- `winception-v2-sbom.cdx.json`
- `LICENSE`

The candidate uses a self-signed internal-test code-signing certificate. Operators must verify the manifest, SHA-256, subject, thumbprint, Code Signing EKU and validity before explicitly using `-TrustSelfSignedCertificate`. A public or organization certificate is deferred.

## Installation

Download every candidate asset into one directory. Do not pipe remote content to PowerShell.

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\Install-Winception.ps1 -Action Check -MsiPath .\Winception-v2.msi -CertificatePath .\Winception-Local-CodeSigning.cer -ReportPath .\check.json
.\Install-Winception.ps1 -Action Install -MsiPath .\Winception-v2.msi -CertificatePath .\Winception-Local-CodeSigning.cer -TrustSelfSignedCertificate -ShowSetupCode -OpenBrowser -ReportPath .\install.json
```

The management UI remains loopback-only during first acceptance. The installer does not change NIC settings or start PXE DHCP.
