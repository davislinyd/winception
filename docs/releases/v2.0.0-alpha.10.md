# Winception 2.0.0-alpha.10 internal candidate

Alpha.10 replaces the unpublished alpha.9 candidate after installed failure injection proved that the default WiX major-upgrade schedule could remove the previous product before a failing health probe was able to roll it back.

The MSI now schedules old-product removal after `InstallInitialize`, inside the Windows Installer rollback transaction. The regression gate requires that exact schedule. The candidate uses MSI `ProductVersion 2.0.25`, `AGPL-3.0-only`, and the explicitly trusted self-signed test certificate. It remains internal and must pass the same 8080 conflict rollback test plus installed v1 migration before release.
