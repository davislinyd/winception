01 · Overview

產品概要

Winception 是 Windows 11 zero-touch deployment 工具。部署主機提供 Web Console 與部署資料服務；目標電腦只需從 UEFI IPv4 PXE 開機，後續 WinPE、OSDCloud、SetupComplete、client software 與 custom scripts 會自動完成。

HOST

部署主機

安裝 Web Console、管理 runtime、OS images、profiles、services 與部署狀態。

CLIENT

目標電腦

從 PXE 進入 WinPE，掛載 SMB share，套用 Windows image，首次開機後完成 finalizer。

CONTROL

安全閘門

所有服務啟動前都要完成 endpoint sync、profile publish 與 preflight。

02 · Host install

部署主機安裝

git clone <repo-url> <repo-root>
cd '<repo-root>'
.\Setup-DeploymentServer.cmd

Setup 會安裝 host management bundle 到 C:\OSDCloud\HostTools\App，並建立可變狀態根目錄 C:\OSDCloud\HostTools\State。它只準備 Web Console 與主機 prerequisites，不會自動啟動 deployment services。

階段做什麼不做什麼
Prerequisites檢查 Git、Node.js、npm、PowerShell modules。不建立 deployment runtime。
Install bundle安裝 App 與 State。不寫入 deployment secrets。
Web launch啟動 http://127.0.0.1:8080。不啟動 DHCP/TFTP/HTTP deployment services。

03 · Services

服務如何運作

Web Console 是控制面；DHCP、TFTP、HTTP、SMB 與 Torrent 是資料面。Endpoint Sync 負責讓 live boot files、WinPE embedded endpoint、SMB firewall 與 local overlay 保持一致。

服務角色
DHCP responder依 DHCP mode 提供 lease 或 PXE boot options。
TFTP service提供 Secure Boot chain 需要的 Microsoft-signed boot files。
HTTP media service提供 iPXE script、WinPE boot files、status API、screenshot API 與 Torrent control API。
SMB share供 WinPE 讀取 selected OS WIM、Apps、Scripts 與 manifests。
Torrent tracker/seeder多台 client 同時部署時分散 OS WIM 傳輸壓力。

05 · Preparation

部署前準備

  1. 確認 deployment project root。
  2. 儲存 deployment secrets。
  3. 執行 Prepare runtime。
  4. 選擇 service interface 與 endpoint。
  5. 在 OS Image Cache 匯出 deployable WIM。
  6. 發布 active deployment profile。
  7. Run preflight 並清除 blocking failures。
  8. 確認 DHCP mode 後啟動 services。

DHCP Server

Winception 管理隔離部署網段的 lease；同一網段不應有其他 DHCP responder。

PXE Proxy

既有網路提供 DHCP，Winception 只提供 PXE boot options；需確認 client 可連回 <service-ip>。

06 · Offline media

USB / ISO 離線模式

手冊原本只描述 PXE active path。若要製作離線安裝媒體，請使用 New-WinceptionUsbInstaller.cmd。它會從目前 active deployment state 匯出 immutable snapshot：selected WIM、active profile 的 Apps / Scripts、目前存在的 driver packs、boot files 與 deployment secrets；不會修改既有 PXE endpoint、services、published boot.wim 或 retired ISO path。

.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3 -CheckOnly
.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3
.\New-WinceptionUsbInstaller.cmd -Iso
.\New-WinceptionUsbInstaller.cmd -Iso -OutputPath D:\Winception-USB.iso
.\New-WinceptionUsbInstaller.cmd -Iso -OpenInRufus -RufusPath C:\Tools\rufus.exe
  • -CheckOnly 只做前置檢查，不寫磁碟。會驗證 merged live config、OSD / OSDCloud modules、boot files、active profile、selected WIM、Apps / Scripts、driver packs、deployment secrets、容量與 FAT32 限制。
  • -Usb 只接受非 boot/system 的 USB disk，執行前會顯示 model、serial、size，並要求精確輸入 ERASE DISK <number>。
  • 輸出 USB 為 GPT、UEFI x64、FAT32 WinPE 與 NTFS OSDCloudUSB；-Iso 則輸出 no-prompt UDF ISO 到 <project-root>\Exports。
  • -OpenInRufus 只預載 ISO 與 NTFS preference；Rufus 仍由 operator 選目標 disk 並開始寫入。
  • 離線 WinPE 只允許一個 internal target disk；同一 media 完整套用後會以 media ID + appliedAt 阻止再次清除。
敏感媒體USB / ISO 內含可部署 Windows 的 local secrets，必須視為敏感資產。這條路徑的證據只證明該次 snapshot，不可替代 PXE readiness。
目前驗收正式離線驗收已完成 ISO 路徑：rebuild 後的 ISO 在 Hyper-V Gen2、Secure Boot ON、無 NIC 的條件下部署到 windows-desktop-ready。

07 · Client deployment

部署一台電腦

  1. 在 Web Console 按 Start all services。
  2. 目標電腦選擇 UEFI IPv4 PXE boot。
  3. WinPE 掛載 SMB share，讀取 selected manifests，套用 Windows image。
  4. Windows 第一次開機後自動登入 <windowsUsername>。
  5. Windows finalizer 執行 active profile 的 software/custom scripts。
  6. Web Console 收到 windows-desktop-ready。
Client 網路邊界WinPE、OSDCloud、OOBE injection 與 SetupComplete staging 只需要連到 Winception server，不需要外部 Internet。外部 Internet 只在 post-logon finalizer 開始執行 software/custom scripts 後才屬於允許範圍；若腳本依賴 BITS 或 Windows Update，腳本需自行啟動必要服務或改用直接下載工具。

01 · Overview

Product Overview

Winception is a Windows 11 zero-touch deployment toolkit. The deployment host provides the Web Console and deployment data services; the target computer only needs to boot from UEFI IPv4 PXE, and WinPE, OSDCloud, SetupComplete, client software, and custom scripts finish automatically.

HOST

Deployment host

Installs the Web Console and manages runtime, OS images, profiles, services, and deployment state.

CLIENT

Target computer

Boots into WinPE from PXE, mounts the SMB share, applies the Windows image, and finishes the finalizer after first boot.

CONTROL

Safety gates

Endpoint sync, profile publish, and preflight must finish before services start.

03 · Services

How The Services Work

The Web Console is the control plane; DHCP, TFTP, HTTP, SMB, and Torrent form the data plane. Endpoint Sync keeps live boot files, the WinPE embedded endpoint, SMB firewall rules, and local overlay aligned.

ServiceRole
DHCP responderProvides leases or PXE boot options according to DHCP mode.
TFTP serviceServes the Microsoft-signed boot files required by the Secure Boot chain.
HTTP media serviceServes iPXE script, WinPE boot files, status API, screenshot API, and Torrent control API.
SMB shareLets WinPE read the selected OS WIM, Apps, Scripts, and manifests.
Torrent tracker/seederDistributes OS WIM transfer load when many clients deploy together.

05 · Preparation

Pre-Deployment Preparation

  1. Confirm the deployment project root.
  2. Save deployment secrets.
  3. Run Prepare runtime.
  4. Select the service interface and endpoint.
  5. Export a deployable WIM in OS Image Cache.
  6. Publish the active deployment profile.
  7. Run preflight and clear blocking failures.
  8. Confirm DHCP mode, then start services.

DHCP Server

Winception manages leases on an isolated deployment network; no other DHCP responder should answer on the same segment.

PXE Proxy

Existing network DHCP provides leases and Winception only supplies PXE boot options; confirm clients can reach <service-ip>.

06 · Offline media

USB / ISO Offline Mode

This manual originally described only the PXE active path. To create offline installation media, use New-WinceptionUsbInstaller.cmd. It exports an immutable snapshot from the current active deployment state: the selected WIM, active profile Apps / Scripts, current driver packs, boot files, and deployment secrets. It does not modify the existing PXE endpoint, services, published boot.wim, or the retired ISO path.

.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3 -CheckOnly
.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3
.\New-WinceptionUsbInstaller.cmd -Iso
.\New-WinceptionUsbInstaller.cmd -Iso -OutputPath D:\Winception-USB.iso
.\New-WinceptionUsbInstaller.cmd -Iso -OpenInRufus -RufusPath C:\Tools\rufus.exe
  • -CheckOnly runs prerequisite checks without writing a disk. It validates merged live config, OSD / OSDCloud modules, boot files, the active profile, selected WIM, Apps / Scripts, driver packs, deployment secrets, capacity, and FAT32 limits.
  • -Usb accepts only non-boot, non-system USB disks. Before writing, it shows model, serial, and size, then requires the exact text ERASE DISK <number>.
  • USB output is GPT, UEFI x64, a FAT32 WinPE partition, and an NTFS OSDCloudUSB partition. -Iso exports a no-prompt UDF ISO to <project-root>\Exports.
  • -OpenInRufus only preloads the ISO and NTFS preference. The operator still chooses the target disk and starts the write in Rufus.
  • Offline WinPE allows one internal target disk only. After a successful deployment, the same media is blocked from wiping the machine again by media ID + appliedAt.
Sensitive mediaUSB / ISO media contains deployable local Windows secrets and must be handled as a sensitive asset. Evidence from this path proves only that specific snapshot and does not replace PXE readiness.
Current validationFormal offline validation is complete for the ISO path: the rebuilt ISO reached windows-desktop-ready on Hyper-V Gen2 with Secure Boot ON and no NIC.

07 · Client deployment

Deploy One Computer

  1. Click Start all services in the Web Console.
  2. Select UEFI IPv4 PXE boot on the target computer.
  3. WinPE mounts the SMB share, reads selected manifests, and applies the Windows image.
  4. Windows automatically signs in as <windowsUsername> on first boot.
  5. The Windows finalizer runs the active profile software/custom scripts.
  6. The Web Console receives windows-desktop-ready.
Client network boundaryWinPE, OSDCloud, OOBE injection, and SetupComplete staging only need to reach the Winception server and do not require external Internet. External Internet is allowed only after the post-logon finalizer starts software/custom scripts; scripts that depend on BITS or Windows Update must start the required services themselves or use a direct download tool.
