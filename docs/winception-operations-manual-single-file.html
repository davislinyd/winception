01 · Product

產品概要

Winception 是 Windows 11 的 zero-touch deployment 工具。部署主機負責準備 runtime、映像與服務端點；目標電腦只需從 UEFI IPv4 PXE 開機，後續 WinPE、OSDCloud、Windows SetupComplete、應用程式與自訂腳本流程會自動完成。

Host

主機責任

安裝 C:\OSDCloud\HostTools\App，用 Web Console 管理 deployment root、service endpoint、OS image、profile 與 preflight。

Client

Client 責任

目標電腦只做 PXE 開機，WinPE 會自動掛載 SMB、套用映像、交接 Windows finalizer。

Guardrails

安全邊界

Git clone 只是 source；live runtime 在 Web 選定的 deployment root。不要直接手改 runtime 檔案。

02 · Host setup

部署主機安裝

git clone <repo-url> <repo-root>
cd "<repo-root>"
.\Setup-DeploymentServer.cmd

Setup-DeploymentServer.cmd 只負責讓 Web Console 與 HostTools 可啟動，不會自動下載 OS image、寫入 secrets、建立 deployment runtime 或啟動 DHCP/TFTP/HTTP。

階段會做不會做
Prerequisites檢查 Git、Node.js、npm、PowerShell modules不建立 deployment runtime
Install bundle安裝 HostTools\App 與 HostTools\State不寫入 deployment secrets
Node setup執行 npm install 並確認主程式可載入不下載 OS image 或 client software
Web launch啟動本機 Web Console不啟動 deployment services

03 · Architecture

服務如何運作

控制面、開機面、映像資料面與狀態面彼此分離。Web Console 是唯一操作入口，但不是 deployable runtime 本身。

04 · Boundaries

Runtime / State / Secrets 邊界

路徑用途管理方式

05 · Readiness

部署前準備

  1. 確認 deployment root，預設可使用 C:\OSDCloud。
  2. 輸入 Deployment Secrets。
  3. 執行 Prepare runtime。
  4. 選定 service endpoint，執行 sync。
  5. 在 OS Image Cache 匯入或下載來源映像，匯出 deployable WIM。
  6. 建立並發佈 active deployment profile。
  7. 執行 Run preflight，直到所有 blocking checks 通過。
  8. 確認 DHCP mode 與 boot mode 正確後，再按 Start all services。

06 · Offline media

USB / ISO 離線模式

.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3 -CheckOnly
.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3
.\New-WinceptionUsbInstaller.cmd -Iso
.\New-WinceptionUsbInstaller.cmd -Iso -OutputPath D:\Winception-USB.iso
.\New-WinceptionUsbInstaller.cmd -Iso -OpenInRufus -RufusPath C:\Tools\rufus.exe
  • -CheckOnly 只做 prerequisite checks，不寫磁碟。
  • -Usb 只能寫非開機、非系統 USB 磁碟，且必須輸入完整確認文字。
  • -Iso 會從當前 active deployment state 匯出不可變快照，不會改 PXE endpoint 或 services。
  • -OpenInRufus 只預載 ISO 與偏好，真正選磁碟與開始寫入仍由操作人員在 Rufus 執行。
敏感媒體 USB / ISO 內含可部署的 local Windows secrets，必須視為 sensitive asset。這條路徑的驗證結果只代表該 snapshot，不等於 PXE readiness。

07 · Client deployment

部署一台電腦

  1. 在 Web Console 按 Start all services。
  2. 在目標電腦選擇 UEFI IPv4 PXE 開機。
  3. WinPE 掛載 SMB share，讀取 selected manifests，套用 Windows image。
  4. Windows 第一次開機後停在 <windowsUsername> 登入畫面；技術人員手動登入後才開始 finalizer。只有 profile 明確啟用的測試專用 automatic sign-in 會略過此步驟。
  5. Windows finalizer 依 active profile 執行 software / custom scripts。
  6. Web Console 收到 windows-desktop-ready 後，才算部署完成。

01 · Product

Product Overview

Winception is a Windows 11 zero-touch deployment tool. The deployment host prepares the runtime, image, and service endpoint; target computers only need to boot from UEFI IPv4 PXE, after which WinPE, OSDCloud, Windows SetupComplete, applications, and custom scripts proceed automatically.

Host

Host responsibility

Install C:\OSDCloud\HostTools\App and use the Web Console to manage the deployment root, endpoint, OS image, profile, and preflight.

Client

Client responsibility

The target only PXE-boots. WinPE mounts SMB, applies the image, and hands off to the Windows finalizer.

Guardrails

Safety boundary

The Git clone is source only; the live runtime lives under the Web-selected deployment root. Do not patch runtime files directly.

02 · Host setup

Deployment Host Installation

git clone <repo-url> <repo-root>
cd "<repo-root>"
.\Setup-DeploymentServer.cmd

Setup-DeploymentServer.cmd only makes the Web Console and HostTools runnable. It does not automatically download an OS image, write secrets, build the deployment runtime, or start DHCP/TFTP/HTTP.

05 · Readiness

Pre-Deployment Preparation

  1. Confirm the deployment root, usually C:\OSDCloud.
  2. Enter Deployment Secrets.
  3. Run Prepare runtime.
  4. Select the service endpoint and sync it.
  5. In OS Image Cache, import or download a source image and export one deployable WIM.
  6. Create and publish the active deployment profile.
  7. Run Run preflight until every blocking check passes.
  8. Only after the DHCP mode and boot mode are confirmed, click Start all services.

06 · Offline media

USB / ISO Offline Mode

.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3 -CheckOnly
.\New-WinceptionUsbInstaller.cmd -Usb -DiskNumber 3
.\New-WinceptionUsbInstaller.cmd -Iso
.\New-WinceptionUsbInstaller.cmd -Iso -OutputPath D:\Winception-USB.iso
.\New-WinceptionUsbInstaller.cmd -Iso -OpenInRufus -RufusPath C:\Tools\rufus.exe
  • -CheckOnly validates prerequisites without writing media.
  • -Usb only accepts non-boot, non-system USB disks and requires the full confirmation text before writing.
  • -Iso exports an immutable snapshot of the current active deployment state and does not modify the PXE endpoint or services.
  • -OpenInRufus only preloads the ISO and preference; the operator still chooses the disk and starts the write in Rufus.
Sensitive media USB / ISO output contains deployable local Windows secrets and must be handled as a sensitive asset. Evidence from this path proves only that snapshot and does not replace PXE readiness.

07 · Client deployment

Deploy One Computer

  1. Click Start all services in the Web Console.
  2. Choose UEFI IPv4 PXE boot on the target computer.
  3. WinPE mounts the SMB share, reads the selected manifests, and applies the Windows image.
  4. On first boot, Windows signs in automatically as <windowsUsername>.
  5. The Windows finalizer runs the active profile software and custom scripts.
  6. The deployment is complete only after the Web Console receives windows-desktop-ready.
